请帮我用 Tauri（v2）开发一个跨平台桌面 PDF 阅读器应用,目标支持 Windows、macOS、Linux 三个平台。

【第一步:项目初始化】
请先在当前文件夹下使用 Tauri 官方命令行工具初始化项目(如 `npm create tauri-app@latest`),前端模板选择 vanilla JavaScript 或 TypeScript(不需要 React/Vue 等框架,这是一个交互逻辑不复杂的工具型应用),应用名称和包标识符可自行合理命名。初始化完成后,使用官方插件添加命令(如 `npm run tauri add dialog`、`npm run tauri add fs`、`npm run tauri add store`)分别引入以下官方插件:

- dialog:用于原生文件选择对话框
- fs:用于读取用户选中的 PDF 文件二进制内容(注意:webview 不能直接通过 file:// 路径加载本地 PDF,必须经 fs 插件读取为字节数组后再交给前端的 PDF.js 解析)
- store:用于本地持久化用户设置(封面模式开关、上次打开文件所在目录等)

并在 src-tauri/capabilities 中正确配置好这几个插件的权限,确保主窗口有权调用。

前端 PDF 解析渲染使用 pdfjs-dist(通过 npm 安装,在 Vite 构建的前端代码中直接 import 使用)。

【核心功能】

1. 双页跨页显示:模拟纸质书的阅读方式。提供"封面模式"开关(界面上有明显的切换按钮,可随时开关,并通过 store 插件记住用户上次的选择,下次启动恢复):
   - 开启时:第 1 页单独占满显示区域,从第 2 页起两两配对显示(2-3、4-5、6-7……),奇数总页数时最后一页单独居中显示。
   - 关闭时:从第 1 页开始就两两配对显示(1-2、3-4、5-6……)。
2. 全屏模式:使用 Tauri 的窗口 API 控制原生全屏(setFullscreen),打开 PDF 后自动进入全屏,背景为纯黑或深灰色。按 ESC 退出全屏,回到正常窗口而不是关闭应用。
3. 翻页交互(以下方式同时支持):
   - 鼠标右键点击页面任意位置 = 下一页(阻止默认右键菜单)
   - 鼠标左键点击屏幕右半区 = 下一页,左半区 = 上一页
   - 键盘方向键(→/↓/PageDown/空格)= 下一页,(←/↑/PageUp)= 上一页
   - 鼠标滚轮向下 = 下一页,向上 = 上一页
4. 文件加载方式:
   - 调用 dialog 插件提供的原生文件选择框
   - 支持将 PDF 文件拖拽到应用窗口直接打开(使用 Tauri 的拖放事件监听)
   - 支持通过命令行参数传入 PDF 路径打开(在 Rust 主进程读取启动参数),为后续做系统文件关联("用此应用打开"PDF)留好接口
5. 自适应缩放:基于 canvas 渲染 PDF.js 页面,根据当前窗口尺寸自动计算缩放比例,使每页等比例缩放后填满可用高度,两页间留一条细分隔线模拟书脊。窗口大小变化或全屏切换时重新计算并重新渲染。
6. 性能优化:翻页时预渲染下一组页面,已渲染页面做缓存,避免重复渲染同一页造成的卡顿。
7. 阅读辅助 UI:屏幕角落显示当前页码/总页数,鼠标静止数秒后自动淡出隐藏,移动鼠标重新显示。

【交互细节要求】

- 翻页动画简洁(fade 或 slide 即可)。
- 未加载 PDF 时显示简洁的拖拽/选择文件提示界面。
- 翻到最前/最后一页时不报错,停留并给轻微视觉反馈即可。
- 整体视觉风格简洁克制,核心是沉浸式阅读体验,不需要复杂工具栏。

【打包配置】
在 tauri.conf.json 中配置好三端打包目标:Windows(nsis 或 msi)、macOS(dmg)、Linux(deb 和 AppImage)。说明清楚实际出包需要在对应操作系统上分别执行 `tauri build`,无法在单一平台上一次性生成全部三端安装包。

请在完成开发后提供清晰的本地运行说明(开发模式启动命令)以及各平台打包命令说明。
